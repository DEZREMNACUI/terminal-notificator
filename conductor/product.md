# Initial Concept

我想要构建一个 rust 编写的 cli 工具,这个工具运行时,可以弹出 mac 提示,功能类似于terminal-notifier,但是我这个工具可以在弹出提示时,点击提示可以跳转到对应的 app.要求启动速度快.

# Product Definition

## Vision
构建一个极致轻量、启动速度极快的 macOS 命令行通知工具。它的核心使命是提供“无缝回归”的体验：当长耗时任务完成并弹出通知时，用户只需点击通知，即可瞬间唤起并跳转回最初执行命令的终端窗口（如 Cursor, Warp, iTerm2 等），无需任何额外配置。

## Target Audience
- **开发人员**: 集成到构建脚本、测试套件或长时间运行的编译任务中。
- **系统管理员**: 用于服务器监控脚本或本地自动化任务的即时反馈。
- **高级用户**: 结合 Shell 脚本增强日常工作流的交互性。

## Core Features
1.  **零配置上下文感知 (Context-Aware Navigation)**:
    -   自动检测运行 CLI 的父进程（终端模拟器）。
    -   点击通知时，精准激活发起请求的终端应用程序窗口，无需用户手动指定 Bundle ID 或应用名称。
2.  **高性能 (High Performance)**:
    -   使用 Rust 编写，确保毫秒级的启动速度和极低的资源占用。
    -   无运行时依赖，单二进制文件分发。
3.  **原生 macOS 通知**:
    -   发送符合系统原生 UI/UX 的通知气泡。
    -   支持标准的标题 (Title) 和内容 (Message)。

## User Stories
- 作为一名开发者，我在 Cursor 中运行一个耗时 5 分钟的构建脚本。构建完成后，我收到一条通知。点击它，系统直接将焦点切回 Cursor 窗口，让我能立即查看日志，而不是仅仅打开一个无关的链接。
- 作为一名运维人员，我在 Warp 中执行数据迁移。我切换到浏览器看文档。当迁移完成的通知弹出时，我点击它，立即回到了 Warp 继续操作。
